🚀 핵심 구현 사항
1. 상태 기반 화면 제어 (ViewMode)
   안드로이드의 프래그먼트 전환 방식에서 영감을 얻어 ViewMode 상태에 따라 화면과 입력 핸들러가 1:1로 매칭되도록 설계했습니다.
   CATEGORY_FILTER와 PRODUCT_LIST 단계를 분리하여 CLI 특유의 메뉴 번호 충돌 문제를 근본적으로 해결했습니다.


2. 등급별 할인 및 누적 시스템 (Enum & UserLevel)
   결제 누적 로직: 고객의 이전 결제 금액에 현재 결제액을 합산하여 실시간으로 고객 정보를 업데이트합니다.
    자동 등급 승급: 누적 금액에 따라 BRONZE(0%), SILVER(5%), GOLD(10%), PLATINUM(15%) 등급이 자동으로 산정됩니다.
    할인율 적용: 등급별 할인율을 UserLevel Enum에 캡슐화하여 비즈니스 로직의 응집도를 높였습니다.
   

3. 스트림 기반 동적 필터링 (Lambda & Stream)
   가격대별 필터링: PriceFilter Enum과 자바 스트림의 filter()를 결합하여 100만 원 이하/초과 상품을 동적으로 조회합니다.
    유연한 확장성: 새로운 필터 조건이 필요할 때 메인 로직을 수정하지 않고 Enum 추가만으로 확장이 가능한 구조(OCP)를 지향했습니다.

🛠 리팩토링 여정 

1단계 (절차지향): 카테고리별로 기능을 단순 나열하여 구현했습니다.

2단계 (입출력 분리): 스파게티 코드를 방지하기 위해 render(출력)와 handle(입력) 메서드를 완전히 분리했습니다.

3단계 (상태 중심 설계): 확장의 한계를 느끼고 ViewMode를 도입, 화면 단위로 입출력을 묶어 관리함으로써 안드로이드 액티비티와 유사한 구조로 진화시켰습니다.
